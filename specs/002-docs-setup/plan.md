## Summary

Set up a complete, bilingual (EN/FR) MkDocs Material documentation site for `apicurio-serdes` with: homepage, quickstart, migration guide, conceptual explanations, how-to guides, auto-generated API reference, changelog, and full-text search in both languages. ## MkDocs Configuration Design

### Updated `mkdocs.yml`

```yaml
site_name: apicurio-serdes
site_description: Python serialization library for Apicurio Registry native v3 API

theme:
  name: material
  language: en
  palette:
    - scheme: default
      primary: indigo
      accent: indigo
      toggle:
        icon: material/brightness-7
        name: Switch to dark mode
    - scheme: slate
      primary: indigo
      accent: indigo
      toggle:
        icon: material/brightness-4
        name: Switch to light mode
  features:
    - navigation.tabs
    - navigation.sections
    - navigation.instant
    - navigation.top
    - navigation.indexes
    - navigation.footer
    - content.code.copy
    - content.tabs.link
    - search.highlight
    - search.suggest
    - toc.integrate

plugins:
  - search:
      lang:
        - en
        - fr
      separator: '[\s\u00a0\-,:!=\[\]()"/]+|(?!\b)(?=[A-Z][a-z])|\.(?!\d)|&[lg]t;'
  - i18n:
      docs_structure: suffix
      languages:
        - locale: en
          default: true
          name: English
          build: true
        - locale: fr
          name: FranГ§ais
          build: true
  - mkdocstrings:
      handlers:
        python:
          paths: [src]
          options:
            docstring_style: google
            show_source: false
            show_root_heading: true
            show_symbol_type_heading: true
            separate_signature: true
            signature_crossrefs: true
            members_order: source

extra:
  alternate:
    - name: English
      link: /
      lang: en
    - name: FranГ§ais
      link: /fr/
      lang: fr

nav:
  - Home: index.md
  - Getting Started:
      - Installation: getting-started/installation.md
      - Quickstart: getting-started/quickstart.md
  - User Guide:
      - Avro Serializer: user-guide/avro-serializer.md
  - Concepts:
      - Wire Format: concepts/wire-format.md
      - Schema Caching: concepts/schema-caching.md
      - Addressing Model: concepts/addressing-model.md
  - How-to Guides:
      - Custom Serialization: how-to/custom-serialization.md
      - Identifier Selection: how-to/identifier-selection.md
      - Error Handling: how-to/error-handling.md
  - Migration:
      - From confluent-kafka: migration/from-confluent-kafka.md
  - API Reference: api-reference/index.md
  - Changelog: changelog.md
```

**Note on nav paths**: The `i18n` plugin with suffix structure expects nav paths without language suffixes. The plugin resolves `index.md` to `index.en.md` or `index.fr.md` based on the build language. The API reference `api-reference/index.md` has no suffix and is shared.

## Edge Case Handling

| Edge Case (from spec) | Resolution |
|------------------------|------------|
| Removed/renamed public symbol in docs | `mkdocs build --strict` with mkdocstrings will fail on broken `::: module.Symbol` references вҖ” caught at build time |
| Untranslated FR section | `mkdocs-static-i18n` falls back to EN content automatically. Untranslated pages display EN content with no broken links. Target: 100% FR coverage before feature completion. |
| Out-of-date code examples | FR-013 validation: code examples should be tested (future CI enhancement). For now, manual review during implementation. |
| Versioned documentation | Deferred (see research.md Decision 7). Single "latest" site for pre-1.0. |
